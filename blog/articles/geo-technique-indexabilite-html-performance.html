GEO technique : indexabilité, HTML et performance

16 février 2026

Indexabilité

Si ton contenu n'est pas explorable par les robots, il est invisible pour les moteurs génératifs. À vérifier :

  • robots.txt : autoriser ou bloquer explicitement les user-agents des IA selon ta stratégie (certains veulent être cités, d'autres non).
  • Pas de noindex / noarchive sur les pages que tu veux voir citer.
  • Pas d'erreurs bloquantes (5xx, contenu derrière auth non prévu pour les bots).

Structure HTML sémantique

Les crawlers des moteurs génératifs découpent le contenu en blocs (chunks). Une hiérarchie claire aide à contextualiser et extraire les bonnes parties.

  • H1 puis H2, H3 : une idée par section.
  • section, article, main : organisation logique du DOM.
  • ul, ol, dl : listes, étapes, définitions.
  • strong, em, blockquote : signaler les passages clés ou citations.
  • Éviter des div sans sens, des imbrications trop profondes, des IDs dupliqués.

Contenu dans le HTML initial (JS)

Contrairement à Google (WRS pour exécuter le JS), beaucoup de moteurs génératifs n'exécutent pas le JavaScript. Si ton contenu est injecté dynamiquement (React, Vue, Angular sans SSR/SSG, accordéons, chargement au scroll), il peut être invisible.

À privilégier : contenu critique présent dans le HTML initial, ou SSR/SSG pour les SPA. Éviter : textes importants uniquement dans des onglets/accordéons JS, blocs chargés en AJAX après interaction.

Performance

Pas de preuve formelle que la vitesse influence directement le GEO, mais un moteur génératif enchaîne requête, crawl, extraction, reformulation. Une page lente ou instable peut être ignorée au profit d'une source plus rapide. Bonnes pratiques : médias compressés (WebP), scripts allégés, contenu principal rendu rapidement, hébergement réactif.
